Comenzi¶

git-branch¶

Gestionează ramurile. De obicei se utilizează pentru:

Afișarea ramurilor existente

git branch

în rezultat se va afișa o listă cu ramurile existente, iar ramura curentă va fi marcată cu ajutorul simbolului *:

  <ramura1>
  <ramura2>
* <ramura3>
  <ramura4>
  <ramura5>

Crearea unei ramuri noi

git branch <ramura nouă>

în rezultat se va crea <ramura nouă> care va avea drept punct de început comiterea curentă. Totodată acestă comandă nu schimbă ramura curentă.

Ștergerea unei ramuri

se realizează folosind opțiunea -d (--delete)

git branch -d <ramură existentă>

Redenumirea unei ramuri

se realizează folosind opțiunea -d (--delete)

git branch -m [<ramura veche>] <ramura nouă>

git-push¶

Încarcă toate modificările operate asupra proiectului local într-un alt proiect aflat de regulă la distanță. Altfel spus realizează sincronizarea unui proiect la distanță cu proiectul local. Cel mai des se utilizează în formatul următor

git push <proiect la distanță> <ramura locală>

Unde <proiect la distanță> poate fi specificat direct prin URL (https://git-scm.com/book/tr/v2/Git-on-the-Server-The-Protocols) sau printr-un nume creat cu ajutorul comenzii git-remote. În rezultat modificările din ramura locală sunt încărcate în ramura cu același nume din proiectul <proiect la distanță>. Dacă ramura de la distanță are alt nume atunci va fi nevoie de schimbat puțin formatul comenzii

git push <proiect la distanță> <ramura locală>:<ramura la distanță>

În cazul când nu este specificată ramura sursă (ramura locală>)

git push <proiect la distanță> :<ramura la distanță>

efectul rulării comenzii este ștergerea ramurii <ramura la distanță>.

Încărcare forțată

Modificările locale pot fi respinse de <proiect la distanță> atunci când acesta conține modificări mai proaspete decât cele locale. În așa caz fie că se integrează noile modificări și apoi se execută încă o dată git-push fie, dacă țineți cu tot adinsul, se suprascriu folosind opțiunea -f (--force)

git push -f <proiect la distanță> <ramura locală>

Warning

În rezultatul încăracării forțate toate modificările mai proaspete decât cele locale vor dispărea din proiectul de la distanță. Din acest motiv asigurați-vă că nu sunt modificări importante pe proiectul la distanță. Printre situațiile când este nevoie de încărcare forțată se numără: Cum să schimb mesajul ultimei comiteri? sau Cum să redenumesc o ramură?.

git-remote¶

Gestionează informațiile privind proiectele la distanță (adăugare, ștergere, vizualizare etc). De regulă în cazul unui singur proiect la distanță (proiect pe server) se folosește în formatul

git remote add origin <URL-ul proiectului>

Pentru a lista alias-urile existente

git remote -v

De exemplu în cazul acestui proiect în rezultatul rulării comenzii de mai sus avem

git remote -v
origin       https://github.com/Streeling/git-rif.git (fetch)
origin       https://github.com/Streeling/git-rif.git (push)
